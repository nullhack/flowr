# ADR-2026-04-26-image-rendering-deferral

## Status

Accepted

## Context

The `image` subcommand would generate an image file from a flow definition. Mermaid syntax is already generated by the domain layer. A rendering tool is needed to convert Mermaid syntax to an image. No mature Python-native library exists that renders Mermaid syntax to images without a browser or Node.js runtime.

## Interview

| Question | Answer |
|---|---|
| Image rendering implementation? | Deferred to v2 — no implementation in v1 |

## Decision

Image generation is deferred to v2. The `image` subcommand will not be implemented.

## Reason

No lightweight Python-native rendering option exists; mmdc requires Node.js; Playwright requires a browser binary (~150MB). Defer until the right solution emerges or the need becomes pressing.

## Alternatives Considered

- **mmdc (mermaid-cli) via subprocess**: standard but requires Node.js runtime
- **Playwright + Mermaid.js**: Python-native API but heavy browser dependency
- **mermaid-py**: Python wrapper around mmdc; still needs Node.js

## Consequences

- (+) No heavy dependencies in v1
- (+) 6 clean subcommands for v1 (validate, states, check, next, transition, mermaid)
- (-) Users cannot generate images from the CLI in v1 (mitigated: mermaid output can be pasted into mermaid.live)
- (-) Two Examples in the feature file need to be marked @deprecated
